# Model Evaluation and Metrics

## Why Evaluation?
학습 도중 중간 평가 -> 학습 조율 위함 (더 나은 학습을 위해)



## Data Splitting
평가는 반드시 **'모델이 한 번도 보지 못한 데이터'**로 해야 한다.<br/>
이를 위해 전체 데이터를 보통 세 그룹으로 나눈다.

* **Train Set** 
  * 모델 학습용 데이터
  * 이 데이터를 보고 수식($a, b$)을 찾는다
* **Validation Set** 
  * 검증용 데이터 : 중간 평가를 통해 학습 방법을 조율한다
* **Test Set:**
  * 시험용 데이터: 처음 보는 데이터로 모델 평가43ㅇㄱㄷ


## Regression Metrics - 선형 회귀 평가 지표
오차가 얼마나 작은가 (얼마나 가깝게 예측했는가) 평가하는 지표

* **MSE (Mean Squared Error)** 
  * 오차를 제곱해서 평균 낸 값 
  * **0에 가까울수록** 좋음 
  * 오차에 제곱을 하기 때문에 큰 실수에 대해 더 엄격한 점수

* **R² (결정계수)**
  - 모델이 데이터의 분산을 얼마나 잘 설명하는지 나타낸다
  - **1에 가까울수록** 좋음   



## Classification Metrics - 로지스틱 회귀 평가 지표
얼마나 정확하게 골라냈는가를 평가하는 지표

* **Accuracy (정확도)** 
    - 전체 중 몇 개나 맞혔는지 보여주는 가장 직관적인 지표

* **F1-score:** 
  * 정답을 맞히는 것(정밀도)과 정답을 놓치지 않는 것(재현율) 사이의 균형 점수
  * 데이터가 불균형할 때 중요

* **AUC (ROC-AUC)** 
  * 모델이 0과 1을 얼마나 명확하게 구분해내는지 '구별 능력'을 수치화한 것
  * **1에 가까울수록** 좋음
