# [BE] Web + Servlet

## 1. 웹 통신의 기본 구조와 프로토콜

* **HTTP 프로토콜** 
  * 브라우저는 HTTP 규약에 따라 헤더와 바디로 구성된 **요청(Request)** 전문을 보낸다.
  
  * 서버는 이를 처리한 후 다시 HTTP 규격에 맞춰 **응답(Response)** 전문을 보낸다.

* **TCP와 계층 구조** 
  * 통신의 연결과 종료는 전송 계층인 **TCP**가 담당

  * 그 위에서 실제 데이터가 오가는 것은 애플리케이션 계층의 역할

* **브라우저의 역할** 
  * 응답받은 HTML을 파싱하여 DOM 트리를 만들고 화면에 렌더링

  * 내부에 포함된 자바스크립트를 실행


## 2. 웹 서버(httpd)와 WAS의 역할 분담
서버는 크게 정적인 콘텐츠를 담당하는 웹 서버와 동적인 로직을 처리하는 WAS로 나뉜다.

* **웹 서버 (httpd)** 
  * 아파치(Apache)와 같은 웹 서버의 핵심 모듈을 **httpd(HTTP Daemon)**라고 부른다.

  * 클라이언트의 요청을 가장 먼저 맞이하며, 미리 저장된 HTML이나 이미지 같은 정적 자원을 즉시 반환한다.

* **WAS (Web Application Server)** 
  * 톰캣(Tomcat)이 대표적이다. 

  * 웹 서버가 처리할 수 없는 동적인 연산(로그인, DB 조회 등)을 뒤편의 애플리케이션에서 실행하도록 돕는다.

* **흐름** 
  * `브라우저 ↔ httpd ↔ WAS ↔ DB` 순

  * httpd는 동적 요청을 WAS에 전달한다.


## 3. 자바 백엔드의 심장: 서블릿(Servlet)
자바에서 동적인 웹 페이지를 만들기 위한 기술이 바로 서블릿이다.

* **제어의 역전(IoC)** 
  * 일반적인 자바 프로그램과 달리 백엔드 코드에는 `main` 함수가 없다.

  * 진짜 `main`은 WAS 내부에 있으며, 개발자가 작성한 클래스를 WAS가 직접 생성하고 실행한다.

  * 이처럼 주도권이 개발자가 아닌 시스템(Container)에 있는 것을 **제어의 역전**이라 한다.

* **서블릿 생명주기** 
    WAS는 서블릿 객체를 관리하며 특정 시점에 메서드를 자동 호출한다.

    1.  `init()`: 객체 생성 시 단 한 번 실행되어 초기화를 수행한다.
   
    2.  `service()`: 요청이 올 때마다 호출되어 실질적인 비즈니스 로직을 처리한다.
   
    3.  `destroy()`: 서버 종료 시 자원을 해제하며 호출된다.


* **URL 매핑** 
    `@WebServlet("/path")` 어노테이션을 통해 브라우저가 어떤 주소로 접속했을 때 해당 서블릿을 실행할지 결정한다.

## 4. JSP는 서블릿이다!
HTML 코드 안에 자바 코드를 넣을 수 있는 JSP(Java Server Pages)는 겉보기엔 문서 같지만 사실 자바 클래스다.

  * 사용자가 `.jsp` 파일을 요청하면 톰캣은 이 파일을 읽어 자바 소스 파일(`.java`)로 번역한 뒤 컴파일하여 클래스 파일(`.class`)을 생성한다.

  * 실제 서버 경로의 `work` 디렉터리를 확인하면 `myjsp_jsp.java`와 같은 파일이 생성되어 있음을 알 수 있다.

  * JSP 역시 메모리상에서는 서블릿 객체로 동작하며, 이를 통해 `HttpServletRequest`와 같은 객체를 주고받으며 동적인 데이터를 처리할 수 있다

## 5. 프레임워크(Spring)로의 진화
프로젝트가 커지면 수많은 서블릿과 객체들을 일일이 관리하기 어려워진다. 이때 등장하는 것이 프레임워크!

* **반제품으로서의 프레임워크** 
  * 프레임워크는 통신, 스레드 관리 등 복잡한 인프라를 미리 만들어둔 것

  * 개발자는 그 틀 안에 비즈니스 로직만 끼워 넣으면 된다!

* **스프링의 확장** 
  * 톰캣이 서블릿, 필터, 리스너 등 몇 가지 규격만 관리했다면

  * 스프링은 개발자가 만드는 **모든 클래스(Bean)**를 컨테이너에 담아 직접 관리하고 연결해준다.

* **프론트 컨트롤러:** 스프링은 모든 요청을 단 하나의 거대한 통로인 `DispatcherServlet`으로 받아 알맞은 하위 컨트롤러에 배분하는 구조를 취한다
